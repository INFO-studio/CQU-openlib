# 说明书

## 站点简介

重庆大学资源共享计划 𝙲𝚀𝚄-𝚘𝚙𝚎𝚗𝚕𝚒𝚋（以下简称 openlib）为重庆大学学生以非官方学生团体身份筹划、发布、更新的资源整合站点，旨在打破信息壁垒，持久化保留一众口口相传的校内相关资源及信息。

openlib 目前由 openlib 运行维护小组进行内容开发、内容更新与请求响应。请见「[贡献者](../../contributor/index.md) - 协作者名单」。

请注意，openlib 及 openlib 运行维护小组均为纯非盈利性质的组织，目前不会开通任何打赏、商业合作、推广服务等渠道。

如您认为您的版权受到了侵犯，请联系 `cqu-openlib@outlook.com`，openlib 运行维护小组会定期检查邮件并给予反馈。

## 开始使用

### 总览

openlib 有若干板块：

- [学业](../../academic/index.md)：获取在重庆大学整个学习生涯的学业相关信息（入学必看、培养方案、转专业等）
- [课程](../../course/index.md)：获取某一课程的具体信息（教材、笔记、试卷等）
- [课表](../../curriculum.md)：查询个人课表
- [社团](../../club/index.md)：获取学校官方对学生开放的社团 / 组织信息
- [技巧](../../skill/index.md)：分享常见技巧（软件安装、环境配置等）
- [生活](../../life/index.md)：获取常见生活方面信息（餐饮、住宿、校园网、校车等）
- [贡献者](../../contributor/index.md)：贡献者名单
- [杂项](../index.md)：说明书、更新日志、待办事项、友情链接等其余内容

您可以进入对应的板块查询相关信息，每个板块均有自己的首页。

**桌面端**

- 顶栏可在各板块之间切换
- 左侧为当前板块的目录
- 中间为正文
- 右侧为「本页」小目录（当页面含有标题时显示）
- 顶栏右侧提供搜索、亮暗色主题切换，以及项目 GitHub 入口

**移动端**

- 点击左上角三杠打开目录抽屉
- 点击目录条目左侧文字：进入目标页
- 点击目录条目右侧箭头：展开下一层目录（若有）

首页上方为「公告」与「您的收藏页」。

### 搜索

点击顶栏「搜索」，或在桌面端按下 `/` 键，即可打开搜索。

搜索可匹配页面标题与课程代码等内容。查找课程时，推荐优先使用课程号，或使用尽量简化的课程名。

由于页面数量较多，搜索有时无法命中目标；此时可改用下文「进入课程页」中的培养方案路径，通常更为可靠。

### 收藏页面

多数文档页在标题旁提供「收藏本页」按钮（已收藏时可「取消收藏」）。首页除外。

在收藏某页后，首页的「您的收藏页」会列出您收藏的全部页面，方便后续再次进入。将常用的培养方案页加入收藏，往往能显著减少找课成本。

收藏功能的数据存储在本地，且不保留页面状态。如您需要保留页面状态或多端同步，请考虑其他相似服务（如浏览器书签）。清除站点数据、更换浏览器或使用无痕模式，可能导致收藏丢失。

### 进入课程页

目前有四种进入课程页查询课程内容的方法。

#### 1. 通过培养方案进入课程页

!!! note "这是进入课程页查询课程内容的最佳实践"

通过 [学业](../../academic/index.md) :l-arrow-right: [专业培养方案](../../academic/专业培养方案/index.md) 的分类进入您专业的培养方案页，再由培养方案中的课程名进入对应课程页。

推荐使用收藏功能，以便快速进入培养方案页，从而快速定位并进入课程页。

#### 2. 通过课表进入课程页

进入 [课表](../../curriculum.md) 板块，按页面提示查询课表后，点击对应课程即可尝试进入课程页。

由于课表返回的结果不可控，有可能会导致跳转进入 404。若遇此情况，请改用培养方案或搜索。

#### 3. 通过搜索进入课程页

见上文「搜索」。推荐使用课程号或最简化的课程名。

由于页面过多，搜索功能有可能不奏效。

#### 4. 通过目录进入课程页

进入 [课程](../../course/index.md) 板块，可通过拼音首字母进行检索。部分课程可能出现在「未分类」中，而未分类列表很长。

课程是否归入字母分类，与课程页是否已有内容并无必然关系。

可以展开未分类部分后，使用浏览器原生查找（Windows：++ctrl+f++ / Mac：++command+f++）搜索对应课程。

## AI 友好

本站正文以 Markdown 编写。任意文档页可在 URL 末尾加上 `.md`，直接获取该页纯文本（`text/markdown`），便于复制、交给 AI 助手或脚本抓取。

**用法：**在正文页路径后追加 `.md`（不要经过 SPA 软跳转，直接打开或请求该地址）：

- 正文 `/academic` → 纯文本 [`/academic.md`](/academic.md)
- 正文 `/sundry/说明书` → 纯文本 [`/sundry/说明书.md`](/sundry/说明书.md)
- 正文 `/curriculum` → 纯文本 [`/curriculum.md`](/curriculum.md)

磁盘上的源文件仍位于 `/doc/...`（例如 `/doc/academic/index.md`），构建时会镜像到与页面同路径的 `.md`，供上述地址访问。

站点根路径另提供 [`/llms.txt`](/llms.txt)，按 [llms.txt](https://llmstxt.org/) 约定列出主要入口。
